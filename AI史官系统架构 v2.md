# AI史官系统架构 v2

## 一、系统目标

AI史官系统的核心目标是：
**基于用户长期积累的生活记录数据，通过结构化分析与语义理解，为用户提供持续性的生活洞察与个性化分析。**

系统需要具备以下能力：

1. **长期数据理解能力**
   能够理解用户跨时间维度的生活记录。

2. **模块化分析能力**
   不同生活领域（如美食、旅行、社交等）可独立分析。

3. **稳定的AI分析结构**
   保证不同问题下AI输出稳定且具备一致结构。

4. **可扩展架构**
   未来可扩展更多分析模块而无需重构系统。

---

# 二、系统整体架构

AI史官系统采用 **四层架构设计**：

```
用户问题层
     │

     │

     │
AI推理层

```


# 五、Prompt构建引擎

Prompt构建由专门模块负责：

```
PromptBuilder
```

结构如下：

```
PromptBuilder
├─ buildSystemPrompt()
├─ buildModulePrompt()
├─ buildStatsPrompt()
├─ buildRecordsPrompt()
└─ buildQuestionPrompt()
```

最终拼接：

```
buildPrompt() =
SystemPrompt
+ ModulePrompt
+ StatsPrompt
+ RecordsPrompt
+ UserQuestion
```

示例伪代码：

```
String buildPrompt() {
  return '''
${buildSystemPrompt()}
${buildModulePrompt()}
${buildStatsPrompt()}
${buildRecordsPrompt()}
${buildQuestionPrompt()}
''';
}
```

这样可以实现：

* 模块独立扩展
* Prompt可维护
* 系统结构清晰

```
AIEngine
├─ buildPrompt()
├─ callModel()
├─ parseResponse()
```

流程：

```
构建Prompt
      │
调用AI模型
      │
接收回复
      │
解析为结构化数据
      │
返回前端
```

伪代码：

```
Future<AIResult> analyze(module, query) {
  prompt = promptBuilder.build(module, query)

  response = ai.call(prompt)

  result = parser.parse(response)

  return result
}
```

# 一、美食模块（Food）

## 1️⃣ 优化后的5个快捷操作Prompt

### ① 口味DNA分析（替代口味偏好分析）

```
请基于我的美食记录数据，深度分析我的“口味DNA”。

不仅要统计喜欢的菜系，还要识别：
1）我真正偏爱的味型（辣/清淡/重油/鲜味等）
2）评分最高的餐厅类型和共性
3）消费价格与满意度之间的关系
4）我更喜欢独自用餐还是社交聚餐
5）是否存在隐藏偏好（如某些菜系评分更高但次数较少）

最后请总结我的美食人格类型，并给出3个符合我口味的探索建议。
```

---

### ② 美食消费模式分析

```
请分析我的美食消费模式，重点包括：

1）消费价格区间分布（低/中/高消费）
2）不同价格区间的满意度差异
3）消费地点分布（城市/商圈）
4）是否存在固定美食路线或高频餐厅类型
5）消费时间规律（工作日/周末、午餐/晚餐）

请总结我的消费习惯，并给出更高性价比的美食策略建议。
```

---

### ③ 高分餐厅幸福因子分析

```
请找出我评分最高的美食记录，并分析这些餐厅让我满意的原因。

重点识别：
1）高评分餐厅的共同特征
2）菜系、价格、环境或服务因素
3）是否与朋友聚餐相关
4）是否存在某些“幸福场景”

最后总结：什么样的餐厅最容易让我感到满足。
```

---

### ④ 我的城市美食版图

```
请根据我的美食记录，构建我的“城市美食版图”。

分析内容包括：
1）各城市的美食记录数量
2）不同城市的偏好菜系
3）我最喜欢的美食城市
4）是否存在某些城市评分更高
5）不同城市的消费差异

最后给出一个属于我的“美食旅行建议地图”。
```

---

### ⑤ 心愿清单打卡策略

```
请分析我的美食心愿清单（wishlist）。

重点分析：
1）心愿餐厅类型
2）城市分布
3）价格区间
4）与我历史口味的匹配度

请帮我制定一个合理的“美食打卡计划”，包括：
优先打卡顺序 + 推荐搭配路线。
```
