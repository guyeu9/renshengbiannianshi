# 旅行模块优化提案

## Context

当前旅行模块存在以下问题：
1. 顶部统计卡片使用硬编码的网络图片，可能导致加载失败
2. 统计数据（"12 个国家，45 座城市"）是硬编码的，不是从数据库实时获取
3. 网络图片加载缺少错误处理和占位符
4. 存在重复的 StreamBuilder 订阅

## Scope

### 涉及文件
- `lib/features/travel/presentation/travel_page.dart` - 旅行模块主页面
- `pubspec.yaml` - 资源配置
- 新增 `assets/images/` 目录存放本地图片

### 不涉及
- 数据库表结构变更
- 架构变更
- 第三方库更换

## Behavior

### 输入
- 用户打开旅行模块页面
- 用户添加/编辑/删除旅行记录

### 输出
1. 顶部统计卡片显示本地图片，不再依赖网络
2. 统计数据（国家/城市数量）从数据库实时计算
3. 网络图片加载时有加载指示器和错误占位图
4. 消除重复的数据库查询，提升性能

## Risks

### 低风险
- 本地图片替换网络图片：不会影响现有功能
- 统计数据实时获取：仅添加数据库查询，不修改写入逻辑
- 图片加载优化：向后兼容，不破坏现有图片显示

### 无风险
- 消除重复 StreamBuilder：纯代码重构，不改变业务逻辑
