# 年度报告系统优化任务分解

## 任务列表

### Task 1: 实现动态年份选择器

**目标**：支持所有有数据的年份 + 当年

**实现步骤**：
1. 创建 `_getAvailableYears()` 方法
2. 从各模块查询有记录的年份
3. 合并当年年份
4. 按降序排列
5. 更新 `_availableYears` getter

**验证方法**：
- 测试有多个年份数据时，下拉列表显示所有年份
- 测试当年即使没有数据也显示
- 测试切换年份后正确加载统计数据

---

### Task 2: 创建终极System Prompt

**目标**：采用架构文档v2的完整System Prompt结构

**实现步骤**：
1. 创建 `UltimateSystemPrompt` 类
2. 实现8大核心章节：
   - 核心角色
   - 分析原则（数据优先、模式发现、趋势分析、避免过度结论）
   - 分析结构（生活洞察、数据观察、行为模式、建议）
   - 数据解释规则（频率、偏好、场景、时间）
   - 分析语气要求（冷静、清晰、观察者视角）
   - 数据不足处理
   - 最终输出格式
   - 目标
3. 提供模块级Prompt构建方法

**验证方法**：
- 检查Prompt包含所有8大章节
- 检查分析原则完整
- 检查输出结构规范

---

### Task 3: 增强统计数据计算

**目标**：新增更多统计维度

**实现步骤**：
1. 扩展 `YearStatistics` 类，新增字段：
   - 月度分布（各模块）
   - 消费区间分布（美食）
   - 场景分布（美食、情绪）
   - 复访率（美食）
   - 心愿达成率（美食）
   - 时段分布（情绪）
   - 星期分布（情绪）
   - 季节分布（旅行）
   - 花费结构（旅行）
   - 分类分布（目标）
   - 顺延统计（目标）
   - 分组分布（羁绊）
   - 联络频率分布（羁绊）
   - 新朋友统计（羁绊）
2. 更新 `_computeStatistics()` 方法，计算新增统计

**验证方法**：
- 测试各模块新增统计字段正确计算
- 测试空数据情况处理
- 测试边界值处理

---

### Task 4: 提取完整原始记录数据

**目标**：传递所有记录的完整信息给AI

**实现步骤**：
1. 创建 `_extractModuleRecords()` 方法
2. 为每个模块提取所有记录：
   - 美食：标题、内容、日期、评分、人均、地点、标签
   - 情绪：内容、日期、心情、场景、颜色
   - 旅行：标题、内容、日期、目的地、花费、地点
   - 目标：标题、备注、总结、分类、进度、完成状态
   - 羁绊：朋友、事件摘要、日期
3. 格式化为Prompt友好的字符串

**验证方法**：
- 测试提取的记录数量与数据库一致
- 测试记录格式正确
- 测试空数据情况处理

---

### Task 5: 实现分批迭代 + 覆盖校验

**目标**：分批处理原始数据，确保所有记录都被处理

**实现步骤**：
1. 创建 `_generateModuleReportWithBatches()` 方法
2. 实现分批逻辑（每批50-80条）
3. 实现覆盖校验：
   - 跟踪已处理的记录ID
   - 检查遗漏
   - 补发遗漏记录
4. 更新进度显示

**验证方法**：
- 测试大批量数据正确分批
- 测试遗漏检测和补发
- 测试进度显示准确

---

### Task 6: 优化报告结构

**目标**：新增生活概览、生活方式、行为模式、偏好画像、趋势变化、AI洞察

**实现步骤**：
1. 扩展 `AnnualReportContent` 类，新增字段：
   - lifeOverview（生活概览）
   - lifestyle（生活方式）
   - behaviorPatterns（行为模式）
   - preferencePortrait（偏好画像）
   - trendChanges（趋势变化）
   - aiInsights（AI洞察）
2. 更新 `_generateReport()` 方法，生成新结构
3. 调整Prompt，要求AI输出新结构

**验证方法**：
- 测试新结构字段正确生成
- 测试AI输出符合新结构
- 测试空字段处理
