# 年度报告系统优化方案提案

## Context

### 现有实现分析

**当前年度报告功能**（位于 `profile_page.dart` 的 `YearReportPage`）：

| 功能 | 状态 | 说明 |
|------|------|------|
| 年份选择器 | ⚠️ 需优化 | 当前仅支持最近5年，需改为支持所有有数据的年份+当年 |
| 统计数据计算 | ✅ 已实现 | YearStatistics 类 |
| 分模块生成 | ✅ 已实现 | 美食/情绪/旅行/目标/羁绊 5个篇章 |
| 进度显示 | ✅ 已实现 | LinearProgressIndicator |
| 报告预览 | ⚠️ 需优化 | 当前仅卡片预览，需支持完整文档预览 |
| PDF导出 | ✅ 已实现 | 使用 pdf 包 |
| 数据库持久化 | ✅ 已实现 | AnnualReviews 表 |

### 问题分析

**1. Prompt过于简单**：
```dart
// 当前实现
systemPrompt: '你是人生编年史APP的年度报告撰写者，擅长用温暖有温度的文字总结美食记忆。'
```
- 缺乏分析原则、输出结构规范
- 没有数据解释规则
- 没有数据不足时的处理逻辑

**2. 统计数据维度不足**：
- 缺少：月度分布、消费区间、场景分布、时间规律
- 缺少：生活节奏分析、偏好画像、社交模式

**3. 报告结构单一**：
- 当前只有6个篇章（开篇+5模块+结语）
- 缺少：生活概览、行为模式、偏好画像、趋势变化、AI洞察

**4. 年份选择器限制**：
- 当前只显示最近5年
- 无法选择有数据的更早年份
- 无法选择当年进行分析

**5. 原始数据传递不足**：
- 当前只传递统计数据
- 需要传递完整原始记录数据

**6. 预览功能不完整**：
- 当前只有卡片式预览
- 需要支持完整文档预览

## Behavior

### 优化方案

#### 1. 升级System Prompt

采用架构文档v2中的终极System Prompt结构：

```
一、核心角色
二、分析原则（数据优先、模式发现、趋势分析、避免过度结论）
三、分析结构（生活洞察、数据观察、行为模式、建议）
四、数据解释规则（频率、偏好、场景、时间）
五、分析语气要求（冷静、清晰、观察者视角）
六、数据不足处理
七、最终输出格式
八、目标
```

#### 2. 动态年份选择器

**实现逻辑**：
1. 查询数据库中所有有记录的年份
2. 合并当年（即使没有数据也显示）
3. 按年份降序排列
4. 动态更新下拉列表

```dart
Future<List<int>> _getAvailableYears() async {
  final db = ref.read(appDatabaseProvider);
  
  // 查询所有有记录的年份
  final years = <int>{};
  
  // 从各模块查询年份
  final foodYears = await db.customSelect(
    'SELECT DISTINCT strftime("%Y", record_date) as year FROM food_records WHERE is_deleted = 0'
  ).get();
  // ... 其他模块
  
  // 合并当年
  years.add(DateTime.now().year);
  
  return years.toList()..sort((a, b) => b.compareTo(a));
}
```

#### 3. 增强统计数据 + 完整原始数据

**统计数据增强**：

| 模块 | 新增统计 |
|------|----------|
| 美食 | 月度分布、消费区间分布、场景分布、复访率、心愿达成率 |
| 情绪 | 月度情绪分布、时段分布、星期分布、场景分布 |
| 旅行 | 季节分布、花费结构、旅行时长分布 |
| 目标 | 分类分布、顺延统计、进度分布 |
| 羁绊 | 分组分布、联络频率分布、新朋友统计 |

**原始数据传递**：
- 传递所有记录的完整信息（标题、内容、日期、评分、地点等）
- 分批次处理（每批50-80条）
- 覆盖校验机制

#### 4. 优化报告结构

**新报告结构**：

```
1. 生活概览
   - 总记录数量
   - 活跃月份
   - 记录类型分布

2. 生活方式
   - 社交型/探索型/规律型

3. 行为模式
   - 周末社交模式
   - 高频餐厅/地点
   - 固定娱乐方式

4. 偏好画像
   - 口味偏好
   - 消费区间
   - 活动类型

5. 趋势变化
   - 今年vs去年对比
   - 行为变化趋势

6. AI洞察
   - 3-5条关键洞察

7. 各模块篇章
   - 美食篇章
   - 情绪篇章
   - 旅行篇章
   - 目标篇章
   - 羁绊篇章

8. 年度结语
```

#### 5. 完整文档预览

**实现方案**：
- 生成报告后，在页面内展示完整文档内容
- 支持滚动浏览所有篇章
- 每个篇章使用卡片样式展示
- 支持折叠/展开

#### 6. 优化生成流程

**分批迭代 + 覆盖校验策略**：

```
Step 1: SQL预聚合统计指标（纯本地）
    ↓
Step 2: 提取模块素材（纯本地）
    - 提取所有原始记录
    - 分批组织数据
    ↓
Step 3: 分模块调AI生成文案（联网API）
    - 分批处理（每批50-80条）
    - 传递完整原始数据
    - 覆盖校验
    - 遗漏补发
    ↓
Step 4: 汇总生成总报告（联网API）
    ↓
Step 5: 本地预览 + PDF导出（纯本地）
```

## Risks

### 潜在风险

1. **生成时间变长**：增加了原始数据传递和校验步骤
   - 缓解措施：优化进度显示，支持取消操作

2. **兼容性**：现有已生成的报告不受影响
   - 缓解措施：新报告使用新结构，旧报告保持原样
