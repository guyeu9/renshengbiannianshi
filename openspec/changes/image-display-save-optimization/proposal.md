# 图片展示与保存优化提案

## Context

当前系统存在以下图片相关问题：

1. **图片展示不完整**：竖拍图片在界面上被横着截断，无法完整显示
2. **图片保存功能缺失**：
   - 点开大图后无法长按进行保存
   - 详情页的点击编辑后，只能进行删除，不能点击图片查看或保存
3. **各模块图片交互不一致**：小确幸、美食、旅行、相遇等模块的图片交互逻辑存在差异

## Scope

涉及文件：

1. **已创建/修改的文件**：
   - `lib/core/utils/image_save_util.dart` - 图片保存工具类
   - `lib/core/widgets/app_image.dart` - 统一图片组件（含SmartImage）
   - `pubspec.yaml` - 已添加image_gallery_saver依赖

2. **需要修复的模块文件**：
   - `lib/features/travel/presentation/travel_page.dart` - 旅行模块图片网格
   - `lib/features/bond/presentation/encounter_pages.dart` - 相遇模块图片组件
   - 各模块详情页的图片网格展示部分

## Behavior

### 1. 图片展示优化

**SmartImage组件**：
- 根据图片宽高比自动选择最佳显示方式
- 竖屏图片（宽高比 < 0.75）使用3:4比例完整显示
- 横屏图片（宽高比 > 1.5）使用16:9比例
- 方形图片使用1:1比例
- 支持点击查看大图、长按保存

### 2. 图片保存功能

**大图预览（ImagePreview）**：
- 单图预览：添加长按保存功能和保存按钮
- 多图画廊：每张图都支持长按保存，添加保存按钮
- 使用ImageSaveUtil处理保存逻辑

**编辑页图片网格**：
- 每个图片项：点击查看大图，长按显示操作菜单（查看/保存/删除）
- 删除按钮：仅删除操作，不与查看/保存冲突
- 支持多图预览画廊

**详情页图片网格**：
- 使用SmartImage组件替换原有实现
- 支持点击查看大图、长按保存
- 支持多图预览画廊

### 3. 各模块统一

统一修复以下模块：
- ✅ 小确幸（moment）模块 - 已完成
- ✅ 美食（food）模块 - 已完成
- ⏳ 旅行（travel）模块 - 待修复
- ⏳ 相遇（encounter）模块 - 待修复
- ⏳ 各模块详情页 - 待修复

## Risks

1. **兼容性风险**：修改现有组件可能影响其他使用这些组件的地方
   - 缓解措施：保持组件API向后兼容，新增功能为可选参数

2. **性能风险**：SmartImage需要先获取图片尺寸，可能增加加载时间
   - 缓解措施：异步加载，不阻塞UI，显示占位符

3. **保存权限风险**：iOS/Android相册保存需要权限
   - 缓解措施：使用image_gallery_saver库自动处理权限请求

4. **用户体验风险**：长按操作可能与其他手势冲突
   - 缓解措施：清晰的视觉反馈，操作菜单提供明确选项
