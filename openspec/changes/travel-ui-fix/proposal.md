# 旅行界面修复提案

## Context

用户报告旅行界面存在多个问题：
1. 点击游记的图片后出现界面错乱
2. 游记详情页无法编辑、无法删除
3. 游记详情页丢失了类似于小确幸详情页的设计
4. 底部的3个按钮不正确
5. 缺少分享功能

经过详细排查，发现以下具体问题：

### 问题1：游记图片点击导航错误
- **位置**：`_buildWechatStyleImages` 函数（`travel_page.dart:3986-4052`）
- **问题**：游记卡片中的图片点击事件导航到了 `TravelDetailPage` 而不是 `JournalDetailPage`
- **影响**：导致用户点击游记图片时看到的是旅行详情页而不是游记详情页，造成界面错乱

### 问题2：游记详情页分享功能未实现
- **位置**：`JournalDetailPage` 底部按钮（`travel_page.dart:6010-6016`）
- **问题**：分享按钮只是显示"分享功能开发中"，没有实际实现
- **参考**：小确幸详情页（`MomentDetailPage`）已实现完整的长图分享功能

## Scope

### 涉及文件
- `lib/features/travel/presentation/travel_page.dart`

### 修改范围
1. 修复 `_buildWechatStyleImages` 函数中的导航逻辑
2. 为 `JournalDetailPage` 实现完整的分享功能（参考小确幸详情页）

## Behavior

### 1. 游记图片点击行为修复
**输入**：用户点击游记卡片中的图片
**输出**：正确导航到 `JournalDetailPage` 而不是 `TravelDetailPage`

### 2. 游记详情页分享功能实现
**输入**：用户点击游记详情页底部的"分享"按钮
**输出**：
- 生成游记详情页的长截图
- 调用系统分享功能
- 用户可以选择分享到其他应用

## Risks

### 低风险
- 仅修改导航逻辑和添加分享功能，不涉及核心业务逻辑
- 分享功能参考已有的小确幸实现，风险可控

### 验证点
- 点击游记图片正确跳转到游记详情页
- 游记详情页编辑、收藏、分享按钮功能正常
- 分享功能可以正常生成截图并调用系统分享
