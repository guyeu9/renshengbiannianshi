# 首页图标统一治理任务拆解

## Task 1：抽离首页统一聚合模型

- 目标：定义首页月历图标与当日日程的统一 DTO 和聚合接口。
- 产出：明确月历输出模型、当日日程输出模型、模块来源声明。
- 验证方法：
  - 审核 DTO 是否覆盖 `dateKey/type/sourceId/title/subtitle/iconName/sortTime/clickTarget/showOnCalendar`
  - 审核每个模块是否都有明确事实源与过滤规则

## Task 2：落地首页聚合层查询入口

- 目标：新增首页统一查询层，替代页面层直接拼装多个流。
- 产出：月历聚合查询入口 + 当日日程聚合查询入口。
- 验证方法：
  - 月历与当日日程不再直接在页面层拼多路业务流
  - 静态检查无新增错误

## Task 3：按模块固化首页显示口径

- 目标：为美食、目标、小确幸、旅行、相遇逐个固化首页生成 / 删除 / 显示规则。
- 产出：模块级规则表与聚合逻辑映射。
- 验证方法：
  - 美食验证心愿清单不进入首页图标
  - 目标验证仅已完成 daily 进入首页
  - 小确幸验证标签图标与隐藏规则符合配置
  - 旅行 / 相遇验证删除后首页同步移除

## Task 4：统一首页 dots / icons / 日程口径

- 目标：消除 dots 与 icons 分裂来源的问题。
- 产出：首页日期格视觉元素统一从聚合层输出。
- 验证方法：
  - 存在首页图标的日期不会出现明显“有图标没点”错位
  - 首页当日日程数量与首页月历状态感知保持一致

## Task 5：补齐回归测试矩阵

- 目标：为首页图标链路建立可持续回归的测试集。
- 产出：覆盖新增 / 编辑 / 删除 / 软删 / 历史补录 / 开关配置的测试用例。
- 验证方法：
  - 通过 `flutter test` 相关测试文件
  - 关键场景至少覆盖：目标完成反转、美食心愿单、小确幸多标签、时间线删除同步

## Task 6：补齐治理文档与 ADR

- 目标：把首页统一治理方案写入正式文档，避免后续再次回到页面层补丁式修复。
- 产出：更新日志、开发设计文档、ADR 决策记录。
- 验证方法：
  - 文档明确写出首页统一口径
  - ADR 中说明为何采用聚合层而不是直接全量 `timeline_events`
