# 旅行详情历史数据与布局错乱修复任务拆解

## Task 1：补全旅行详情页时间线输入数据

### 目标

修复 `TravelDetailPage` 向 `_TravelTimeline` 传参不完整的问题，确保时间线能够拿到游记渲染所需的关联数据。

### 开发任务

1. 审查 `travel_detail_provider.dart` 当前输出字段，确认是否缺少 `entityLinks` 或等价聚合结果。
2. 调整 `TravelDetailState` 的结构，使旅行详情页能拿到时间线渲染所需的 links 数据。
3. 修改 `TravelDetailPage`，将真实 links 传入 `_TravelTimeline`，移除当前空数组占位传参。
4. 检查时间线内部朋友、美食、游记卡片的聚合与排序逻辑，确认补传数据后不会重复渲染。

### 验证方法

```md
- 打开包含游记、朋友关联、美食关联的旅程详情页
- 确认时间线中游记卡片正常显示
- 确认同行者卡片与美食卡片能随关联关系正常出现
- 确认页面不再出现“只有标签、正文区域空白”的现象
```

## Task 2：统一顶部天数与时间线 Day 展示规则

### 目标

修复旅行详情页顶部总天数与时间线 Day 编号口径不一致的问题，避免出现明显错位。

### 开发任务

1. 梳理当前顶部天数、头部日期、时间线 Day 编号分别使用的字段与算法。
2. 确定统一规则，优先保证用户理解一致性，并兼容历史数据缺天、补录、跨天的场景。
3. 调整 `travel_detail_provider.dart` 或 `travel_page.dart` 中相关计算逻辑。
4. 检查空 trip、单日旅行、多日旅行、游记日期不连续等场景下的降级显示。

### 验证方法

```md
- 构造 1 天、3 天、7 天行程数据分别进入详情页
- 构造游记日期不连续的数据，确认顶部天数和 Day 展示仍然合理
- 确认单日旅行显示“1 天”且时间线无错位
- 确认多日旅行不会出现顶部 7 天但时间线只像 3 天的误导性展示
```

## Task 3：统一旅行图片解析与封面回退逻辑

### 目标

统一旅行模块在详情页、卡片、游记图片区的图片解析逻辑，兼容历史数据与新建数据，消除图片堆顶和网格错乱问题。

### 开发任务

1. 排查 `travel_page.dart` 中所有与旅行图片解析相关的函数和调用点。
2. 对齐 `travel_detail_provider.dart` 中历史兼容解析逻辑，避免同一字段被多套规则解析。
3. 补充主行程无图、游记有图、历史逗号分隔、JSON 数组、单图路径等场景的处理。
4. 检查图片网格、头图封面、游记卡片缩略图在统一解析后的显示稳定性。

### 验证方法

```md
- 使用历史逗号分隔图片数据进入旅程详情页，确认图片正常显示
- 使用 JSON 数组图片数据进入旅程详情页，确认图片正常显示
- 使用主行程无图但游记有图的数据，确认封面与内容区显示合理
- 确认游记图片不再浮到页面顶部，不覆盖头图与标签卡片
```

## Task 4：补充旅行详情历史数据回归测试

### 目标

为本次问题补齐自动化测试，覆盖历史数据、缺失 links、时间线分组和图片兼容场景，防止问题再次回归。

### 开发任务

1. 扩展 `travel_detail_provider_test.dart` 或新增针对旅行详情的测试文件。
2. 为 links 聚合补充测试，验证主行程与游记关联能够正确汇总。
3. 为历史图片格式兼容补充测试，验证逗号分隔与 JSON 数组都能正确解析。
4. 为时间线 Day 与头部天数口径相关逻辑补充测试。

### 验证方法

```md
- 运行旅行详情相关 provider/widget 测试并全部通过
- 确认新增测试能覆盖历史图片格式兼容场景
- 确认新增测试能覆盖缺失 links 后恢复正常的场景
- 确认新增测试能覆盖多天与缺天游记场景
```
