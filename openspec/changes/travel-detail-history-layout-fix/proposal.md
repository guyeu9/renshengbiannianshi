# 旅行详情历史数据与布局错乱修复提案

## Context

用户反馈从旅行列表进入每一个旅程详情页后，界面出现明显错乱：

1. 游记内容没有正确展示
2. 天数没有正确展示
3. 时间线几乎没有内容
4. 图片反而堆叠显示在页面上方，遮挡头图与正文

结合现有代码、更新日志和开发设计文档排查后，当前问题不是单一 UI 样式错误，而是由多处数据口径不一致和渲染输入缺失共同导致：

### 已确认的高概率根因

1. **详情页没有把关联数据 links 传给时间线组件**
   - `TravelDetailPage` 当前传给 `_TravelTimeline` 的 `links` 是空数组。
   - `_TravelTimeline` 内部却依赖 `links` 计算游记关联的朋友和美食卡片。
   - 结果会导致时间线内容不完整、顺序异常，用户感知为“游记内容没显示”。

2. **顶部总天数与时间线 Day 编号使用了两套不同口径**
   - 顶部天数来自 `trip.startDate/endDate` 或主记录日期。
   - 时间线 Day 编号来自“有游记的日期分组索引”。
   - 历史数据一旦存在缺天、补录、跨天编辑，就会出现顶部显示 7 天，但时间线只显示第 1/2/3 天的错位现象。

3. **图片字段兼容逻辑在不同层实现不一致**
   - `travel_detail_provider.dart` 中 `_decodeStringList` 已兼容 JSON 数组、逗号分隔和单路径。
   - 但 `travel_page.dart` 中部分入口与卡片逻辑仍存在旧式解析或与详情页不同的解析方式。
   - 历史数据与新建数据混用时，容易导致图片首图异常、网格错位，甚至图片模块顶到页面上方。

4. **历史修复只覆盖了部分路径，没有统一到详情页完整数据链路**
   - 更新日志中已经多次记录旅行详情页、游记图片跳转、历史图片格式兼容等修复。
   - 但从当前代码看，页面入口、Provider 聚合、时间线渲染、图片解析之间仍没有形成统一口径，因此旧问题会反复出现。

## Scope

### 涉及文件

- `life_chronicle/lib/features/travel/presentation/travel_page.dart`
- `life_chronicle/lib/features/travel/providers/travel_detail_provider.dart`
- `life_chronicle/test/providers/travel_detail_provider_test.dart`
- `life_chronicle/test/` 下与旅行详情/时间线相关的新增或补充测试文件
- `更新日志.md`
- `开发设计文档.md`

### 修改范围

1. 修复旅行详情页向时间线传参不完整的问题
2. 统一旅行详情页顶部天数与时间线 Day 展示口径
3. 统一旅行图片字段解析逻辑，兼容历史数据与新数据
4. 补齐针对历史数据、缺失 links、跨天游记分组的自动化验证
5. 在更新日志和开发设计文档中追加本次修复记录

## Behavior

### 输入场景

1. 用户点击旅行列表中的任意旅程进入详情页
2. 详情页加载主行程、游记、关联朋友、关联美食、心愿清单、图片
3. 历史数据可能包含以下情况：
   - 图片为 JSON 数组字符串
   - 图片为逗号分隔字符串
   - 主行程无图但游记有图
   - 行程总天数与游记记录日期不连续

### 预期输出行为

1. 旅程详情页头图、标题、地点、日期、总天数展示稳定且不被游记图片覆盖
2. 时间线正确渲染游记内容、同行者、美食等卡片，不再因空 `links` 导致内容缺失
3. Day 展示规则对用户可理解，至少与顶部天数口径保持一致，不再出现明显错位
4. 历史数据与新建数据的图片都能正常解析和展示，不再出现图片顶到页面顶部或网格异常
5. 页面在旧数据、空数据、部分缺失数据下仍可正常降级展示

## Risks

### 主要风险

1. **时间线 Day 规则调整可能影响现有视觉预期**
   - 如果将 Day 改为基于行程起始日推导，现有测试和部分截图需要同步更新。

2. **图片解析统一后可能影响旅行列表、详情页、游记卡片的封面选择逻辑**
   - 需要验证主行程封面、游记图片网格、历史数据首图回显是否一致。

3. **补传 links 后时间线内容会比当前更多**
   - 需要确认这是恢复正确行为，而不是引入新的重复展示。

### 验证重点

1. 历史数据详情页加载后不再出现图片覆盖头图与正文
2. 游记内容能够在时间线中正常展示
3. 顶部天数与时间线 Day 展示不再互相冲突
4. 朋友、美食关联卡片能够正确跟随游记展示
5. 现有旅行详情相关测试继续通过，并新增历史数据回归测试
