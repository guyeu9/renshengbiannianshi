# 目标模块 UI 重构与功能完善 - Product Requirement Document

## Overview
- **Summary**: 对目标模块进行全面的 UI 重构和功能完善，包括：整体看板布局调整、目标详情页新增复盘和顺延记录展示、右上角操作菜单完善、顺延计划界面新建、目标拆解维护界面重新设计。
- **Purpose**: 解决目前目标界面 UI 不够美观、功能不完整的问题，确保所有数据（复盘内容、顺延记录）不丢失，并使 UI 与原型图保持一致。
- **Target Users**: 人生编年史 APP 的所有目标模块用户。

## Goals
- 目标看板整体布局按原型图重新设计，改善视觉体验
- 目标详情页展示阶段复盘内容和顺延记录
- 目标详情页右上角操作菜单包含：拆解维护、编辑、删除三个按钮
- 新建顺延计划界面，支持数据落库
- 目标拆解维护界面按原型图重新设计，UI 统一

## Non-Goals (Out of Scope)
- 不改变现有的数据库架构（除非新增字段是必要的）
- 不重构其他模块的功能
- 不涉及 AI 功能的改动

## Background & Context
当前目标模块存在以下问题：
1. 整体看板的详细计划进度展示非常紧凑，不够美观，与原型图不一致
2. 目标详情页缺少阶段复盘内容和顺延记录的展示
3. 右上角操作按钮功能不完整，缺少编辑和删除功能
4. 顺延计划界面需要新建并妥善处理数据落库
5. 目标拆解维护界面需要按原型图重新设计

已有原型设计文件位于：
- `前端/stitch_home_schedule_dashboard/5-1目标-人生目标看板/`
- `前端/stitch_home_schedule_dashboard/5-3 目标-目标拆解详情/`
- `前端/stitch_home_schedule_dashboard/5-5 目标 目标拆解维护/`
- `前端/stitch_home_schedule_dashboard/5-6 目标--顺延目标计划/`

## Functional Requirements

### FR-1: 目标看板布局重构
- 按照原型图重新设计目标看板的整体布局
- 改善卡片间距、排版，使展示更加美观舒适
- 保持现有功能逻辑不变

### FR-2: 目标详情页新增内容展示
- 在目标截止时间下方展示阶段复盘内容
- 在目标截止时间下方展示顺延记录
- 确保所有历史数据完整展示，不丢失

### FR-3: 目标详情页右上角操作菜单
- 点击右上角按钮弹出 3 个操作按钮菜单
- 菜单包含：拆解维护（当前编辑按钮界面）、编辑（复用新建目标界面）、删除（删除此目标）
- 操作菜单在按钮下方弹出，样式符合 UI 设计规范

### FR-4: 顺延计划界面新建
- 按原型图新建顺延计划界面
- 支持调整截止日期、填写顺延原因与复盘
- 展示过往顺延记录
- 实现数据落库功能

### FR-5: 目标拆解维护界面重新设计
- 按原型图重新设计目标拆解维护界面
- 确保 UI 与整体设计风格统一
- 支持阶段拆解、任务管理、拖拽排序
- 处理好数据存储逻辑

## Non-Functional Requirements
- **NFR-1**: 所有 UI 必须符合 UI设计规范.md 中的规范
- **NFR-2**: 保持现有数据兼容性，不丢失任何历史数据
- **NFR-3**: 确保界面响应流畅，无明显卡顿
- **NFR-4**: 所有新增功能必须有完整的验证测试

## Constraints
- **Technical**: 必须使用现有的 Flutter + Riverpod + Drift 技术栈
- **Business**: 禁止擅自改架构/换库/改表结构（除非先询问用户）
- **Dependencies**: 依赖现有的数据库表结构

## Assumptions
- 原型图中的设计是用户最终期望的样子
- 数据库现有表结构可以满足新增功能需求（可能需要新增字段）
- 用户希望保持现有功能逻辑不变，只改进 UI 和完善功能

## Acceptance Criteria

### AC-1: 目标看板布局符合原型图
- **Given**: 用户打开目标看板
- **When**: 查看年度目标列表
- **Then**: 布局、间距、卡片样式与原型图一致，视觉效果美观舒适
- **Verification**: `human-judgment`
- **Notes**: 对比原型图 `screen.png`

### AC-2: 目标详情页展示复盘内容
- **Given**: 用户打开有复盘记录的目标详情页
- **When**: 滚动查看页面内容
- **Then**: 在截止时间下方可以看到阶段复盘内容
- **Verification**: `programmatic`
- **Notes**: 确保历史复盘数据完整展示

### AC-3: 目标详情页展示顺延记录
- **Given**: 用户打开有顺延记录的目标详情页
- **When**: 滚动查看页面内容
- **Then**: 在截止时间下方可以看到完整的顺延记录列表
- **Verification**: `programmatic`
- **Notes**: 确保所有顺延历史不丢失

### AC-4: 右上角弹出完整操作菜单
- **Given**: 用户在目标详情页
- **When**: 点击右上角操作按钮
- **Then**: 在按钮下方弹出包含"拆解维护"、"编辑"、"删除"三个按钮的菜单
- **Verification**: `programmatic`

### AC-5: 顺延计划界面可用
- **Given**: 用户点击"顺延计划"按钮
- **When**: 进入顺延计划页面
- **Then**: 可以看到完整的顺延计划界面，包含日期调整、原因填写、过往记录展示
- **Verification**: `programmatic`

### AC-6: 顺延数据正确落库
- **Given**: 用户在顺延计划页面填写信息并确认
- **When**: 点击"确认顺延"按钮
- **Then**: 顺延记录正确保存到数据库，目标截止时间更新，可在详情页看到新的顺延记录
- **Verification**: `programmatic`

### AC-7: 目标拆解维护界面符合原型
- **Given**: 用户点击"拆解维护"按钮
- **When**: 进入目标拆解维护页面
- **Then**: 界面布局、组件样式与原型图一致，UI 风格统一
- **Verification**: `human-judgment`

### AC-8: 目标拆解数据正确存储
- **Given**: 用户在拆解维护页面进行操作
- **When**: 添加/编辑/删除阶段或任务，点击保存
- **Then**: 所有变更正确保存到数据库
- **Verification**: `programmatic`

### AC-9: 删除功能正常工作
- **Given**: 用户在目标详情页点击删除
- **When**: 确认删除操作
- **Then**: 目标被软删除，不再出现在列表中，但数据保留
- **Verification**: `programmatic`

### AC-10: 编辑功能复用新建界面
- **Given**: 用户在目标详情页点击编辑
- **When**: 进入编辑页面
- **Then**: 显示新建目标界面，但预填充当前目标的数据
- **Verification**: `programmatic`
