## 2026-03-05 导出服务编译错误修复

### 📤 导出模块

#### 问题背景
导出服务（Excel/PDF）出现编译错误，主要是 Drift 导入问题、Excel 包 API 兼容性问题、PDF 包 API 兼容性问题和数据库字段名称不匹配问题。

#### 核心修复

**1. Drift 导入修复**
- `excel_export_service.dart`: 将 `import 'package:drift/drift.dart' as drift;` 改为 `import 'package:drift/drift.dart';`
- `pdf_export_service.dart`: 同样修复导入方式
- 原因：`isBiggerOrEqualValue`、`isSmallerOrEqualValue` 等是 Drift 的扩展方法，需要不带前缀的导入才能正确使用

**2. Excel 包 API 兼容性修复**
- 移除所有 `const` 关键字（TextCellValue、CellStyle、IntCellValue、DoubleCellValue 构造函数不是 const）
- 修复 CellStyle 的 backgroundColorHex 参数格式（使用 `#RRGGBB` 格式字符串）
- 保留完整的导出功能：数据概览、各模块工作表、时间范围筛选

**3. PDF 包 API 兼容性修复**
- 移除所有 `const` 关键字（pw.SizedBox、pw.EdgeInsets 等构造函数不是 const）
- `PdfColor` 没有 `withOpacity` 方法，改用 `PdfColor.fromInt()` 计算半透明颜色
- 修复所有 PDF 组件的构造方式

**4. 数据库字段名称修正**
- `FriendRecord` 没有 `relationship` 字段 → 改用 `groupName`
- `TravelRecords` 表没有 `startDate`/`endDate` 字段 → 改用 `recordDate` 和 `planDate`
- `TravelRecord` 没有 `description` 字段 → 改用 `content`
- `GoalRecord` 没有 `status`/`deadline` 字段 → 改用 `isCompleted`/`dueDate`
- `TimelineEvent` 没有 `type`/`description` 字段 → 改用 `eventType`/`note`
- `MomentRecord.content` 是 nullable → 添加 `?? ''` 默认值
- `MomentRecord.mood` 不是 nullable → 移除不必要的 null 检查
- `impressionTags` 是 `String?` 类型而不是 `List`，移除 join 调用

**5. backup_service.dart 修复**
- 移除未使用的 `snapshotPath` 变量声明

#### 涉及文件
| 文件 | 变更类型 | 说明 |
|------|----------|------|
| excel_export_service.dart | 重写 | Drift导入、Excel API兼容性、字段名称修正 |
| pdf_export_service.dart | 重写 | Drift导入、PDF API兼容性、字段名称修正 |
| backup_service.dart | 修复 | 移除未使用变量 |

#### 验证测试

## 2026-03-05 Android 15 地图空白问题诊断与修复

### 🗺️ 地图模块

#### 问题背景
Android 15 设备上地图显示空白，只看到蓝色定位图标，地图区域无内容，显示有定位权限在申请，但是没有任何进度。

#### 核心问题分析与修复

**1. 添加详细的诊断日志**
- 在 `amap_location_page.dart` 的 build 方法中添加完整的地图初始化日志
- 输出 Android 版本、地图选择逻辑、Key 配置状态等关键信息
- 使用统一的日志前缀 `[AmapLocation]`、`[AmapNative]`、`[AmapWebView]`

**2. 增强 WebView 地图诊断**
- 在 `amap_webview_map.dart` 的每个关键方法添加详细日志
- 新增 `initState` 日志，输出初始参数配置
- 新增 `_initLocationPlugin` 日志，追踪定位插件初始化
- 新增 `_requestLocationPermission` 详细日志，输出权限状态变化
- 新增 `_initWebView` 日志，追踪 WebView 初始化流程
- 新增 `_handleLocationRequest` 完整流程日志
- 新增 `onHttpError` 回调，捕获 HTTP 错误
- 统一使用 `[WebView Console]` 前缀输出 JavaScript 日志

**3. 增强原生地图错误处理和降级机制**
- 新增 `_nativeMapFailed` 状态变量，标记原生地图是否失败
- 新增 `_nativeMapCreatedTime` 变量，记录原生地图创建时间
- 新增 `_checkNativeMapTimeout()` 方法，10秒超时检测
- 修改 `canUseNativeMap` 逻辑，考虑 `_nativeMapFailed` 状态
- 修改 `useWebViewMap` 逻辑，原生地图失败时自动降级到 WebView
- 在 `onMapCreated` 回调中记录创建时间
- 添加降级机制日志，清晰记录降级触发过程

  - **根因**：

    - 用户期望完成的小目标能够自动出现在首页日程时间线中
    - 取消完成时应该从时间线中移除该小目标
  - **解决**：

      - 在函数中添加对小目标（level='daily'）的特殊处理
      - 当小目标被标记为完成（isCompleted=true）时：
        - 使用 `insertOnConflictUpdate` 向 `timeline_events` 表插入一条记录
        - 记录内容：id（小目标id）、title（小目标标题）、eventType（'goal'）、startAt（小目标recordDate）、recordDate（小目标recordDate）、createdAt（now）、updatedAt（now）
        - 使用 `insertOnConflictUpdate` 防止重复插入
      - 当小目标被取消完成（isCompleted=false）时：
        - 从 `timeline_events` 表中删除对应的记录
        - 通过小目标的 id 进行匹配删除，同时筛选 eventType='goal'
      - 确保只有 level='daily' 的小目标才执行上述逻辑，大目标不处理
  - **验证**：

    - 取消勾选小目标完成时，该小目标会从时间线中移除
    - 大目标（level!='daily'）的完成状态变更不会影响时间线
    - GetDiagnostics 检查通过，无诊断错误

## 2026-02-25 Android 15 地图搜索与定位修复

### 🗺️ 地图模块
- **Android 15 地图搜索失败与定位按钮无响应修复**：
  - **问题 1：搜索失败 `userkey-plat-nomatch`**
    - **根因**：高德 API 有两种 Key 类型：
      - **Web 服务 Key**：用于 REST API 调用（如地点搜索 `/v3/place/text`）
      - **JS API Key**：用于 WebView 加载 Web 地图
      - 代码中两个场景混用了同一个 Key，导致搜索接口返回 `userkey-plat-nomatch` 错误
    - **解决**：
      - 新增 `_amapJsKey` 环境变量，与 `_amapWebKey` 分离
      - `_amapWebKey` 用于 REST API 搜索（更新为用户提供的 Web 服务 Key `76e66f23c7045fbe296f9aa9b7e7f12c`）
      - `_amapJsKey` 用于 WebView 地图加载（使用 JS API Key `fbe9be9664c1c2b31f7783a10ec61bc8`）
  - **问题 2：WebView 地图无法滑动**
    - **根因**：`webview_flutter` 的 `WebViewWidget` 默认不处理手势事件，在 `ListView` 等滚动容器中无法滑动
    - **解决**：为 `WebViewWidget` 添加 `gestureRecognizers` 参数，注册 `VerticalDragGestureRecognizer` 和 `HorizontalDragGestureRecognizer`
  - **问题 3：定位按钮无响应**
    - **根因**：WebView 中的 `navigator.geolocation` 需要 Android 系统层面的权限处理，WebView 本身不会自动请求 Android 定位权限
    - **解决**：
      - 使用 Flutter 原生定位插件 `amap_flutter_location` 替代 WebView geolocation
      - 在定位按钮点击时通过 Flutter 层请求权限并获取位置
      - 通过 JavaScript Channel 将位置传递给 WebView 地图
      - 添加定位状态 UI（加载中/成功/失败）
      - 添加定位超时处理（15秒超时）
  - **变动文件**：
    - [amap_location_page.dart](file:///d:/trae/chronicle-of-life/life_chronicle/lib/core/widgets/amap_location_page.dart)：新增 `_amapJsKey`，更新 `_amapWebKey`

### Flutter Gradle 配置决策说明（2026-02-23，第一次决策）

#### 背景
在 Flutter 3.27 + Gradle 8.14 + AGP 8.2.2 环境下，新版声明式 `plugins {}` DSL 与旧版命令式 `apply plugin`/`apply from` 方式存在兼容性问题，导致多次构建失败。

#### 新旧配置对比

| 维度 | 新版配置（声明式） | 旧版配置（命令式） |
|------|-------------------|-------------------|
| **语法** | Kotlin DSL (`settings.gradle.kts`/`build.gradle.kts`) | Groovy DSL (`settings.gradle`/`build.gradle`) |
| **插件声明** | `plugins { id("dev.flutter.flutter-plugin-loader") version "1.0.0" }` | `apply from: "$flutterSdkPath/packages/flutter_tools/gradle/app_plugin_loader.gradle"` |
| **Flutter 集成** | 自动应用 `flutter-gradle-plugin` | 手动 `apply from: "$flutterRoot/packages/flutter_tools/gradle/flutter.gradle"` |
| **插件应用** | 声明式，Gradle 自动解析 | 命令式，按顺序执行 |
| **兼容性** | Flutter 4.0+ 推荐 | Flutter 3.x 稳定 |

#### 遇到的问题
1. **`flutter-plugin-loader` 自动应用 `flutter-gradle-plugin`**：若同时显式声明 `flutter-gradle-plugin`，会触发 `Cannot run Project.afterEvaluate(Closure) when the project is already evaluated` 错误
2. **`flutter {}` DSL 未定义**：移除显式 `flutter-gradle-plugin` 后，`flutter {}` 配置块找不到定义
3. **Kotlin DSL 编译失败**：在 Java 24 环境下，Kotlin DSL 脚本解析出现兼容性问题
4. **CI 环境路径问题**：`local.properties` 在 CI 不存在，需要 `FLUTTER_ROOT` 环境变量兜底

#### 最终决策
采用旧版 Groovy DSL 配置，原因：
1. **稳定性优先**：旧版配置经过长期验证，与 Flutter 3.27 兼容性更好
2. **问题隔离**：避免新版声明式 DSL 的隐式行为带来的调试困难
3. **CI 友好**：Groovy 脚本对环境差异容忍度更高

#### 迁移时机
建议在以下条件满足时考虑迁移到新版配置：
- Flutter 升级到 4.0+
- Gradle 升级到 9.0+
- AGP 升级到 9.0+
- 官方明确推荐 Kotlin DSL 配置

### Flutter Gradle 配置迁移说明（2026-02-23，第二次决策：执行迁移）

#### 背景
Flutter 官方弃用警告明确指出旧版 `apply from`/`apply plugin` 方式将在未来版本移除，经分析后决定立即迁移到新版声明式 `plugins {}` 块。

#### 迁移内容

| 文件 | 旧版配置 | 新版配置 |
|------|---------|---------|
| **settings.gradle** | `apply from: "...app_plugin_loader.gradle"` | `pluginManagement {} + plugins { id "dev.flutter.flutter-plugin-loader" version "1.0.0" }` |
| **build.gradle** | `buildscript { dependencies { classpath ... } }` | 移除 `buildscript` 块，仅保留 `allprojects` 仓库 |
| **app/build.gradle** | `apply plugin: "com.android.application"` + `apply from: "...flutter.gradle"` | `plugins { id "com.android.application"; id "dev.flutter.flutter-gradle-plugin" }` |

#### 迁移收益
1. **消除弃用警告**：构建输出更清晰，无警告干扰
2. **未来兼容性**：避免未来 Flutter 版本移除旧版支持导致构建失败
3. **官方推荐**：符合 Flutter 3.16+ 最佳实践

#### 注意事项
- 新版配置中 `flutter-plugin-loader` 会自动应用 `flutter-gradle-plugin`，无需手动 `apply from: "...flutter.gradle"`
- `flutter {}` DSL 由 `flutter-gradle-plugin` 提供，配置块保持不变
- CI 环境需确保 `FLUTTER_ROOT` 环境变量正确设置
- 本地插件（如 `amap_flutter_location`）的 `build.gradle` 也需使用 `plugins {}` 块

- [build.gradle](file:///d:/trae/人生编年国际/life_chronicle/android/build.gradle)：移除 `buildscript` 块，仅保留 `allprojects` 仓库配置
- [app/build.gradle](file:///d:/trae/人生编年国际/life_chronicle/android/app/build.gradle)：使用 `plugins {}` 块应用 `com.android.application`/`kotlin-android`/`dev.flutter.flutter-gradle-plugin`
- [amap_flutter_location/build.gradle](file:///d:/trae/人生编年国际/life_chronicle/plugins/amap_flutter_location/android/build.gradle)：使用 `plugins {}` 块应用 `com.android.library`
